Detailed Description

LotteryService encapsulates the core lottery selection workflow.

Responsibilities:

  • Load eligible waiting-list entries for an event
  • Randomly select up to a requested number of entries
  • Persist selection by updating status/timestamps in batch via WaitingListManager
  • Optionally apply a configurable per-draw response deadline (hours)

This class is UI-agnostic and suitable for unit testing by mocking the underlying WaitingListManager if desired.

int slots,
@Nullable Integer responseWindowHoursOverride,
OnLotteryComplete callback )

Runs a lottery for a given event by sampling up to slots entrants currently in the "waiting" status. Selected entrants are updated to "selected" with timestamps.

If responseWindowHoursOverride is provided, it is written onto each selected entry (field: response_window_hours). Otherwise, downstream logic should fall back to the event's default response window.
